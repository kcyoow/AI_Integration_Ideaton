

# 배경

지금은 일단 상단 네비게이션바에 로그인 버튼만 만들어 놓은 상태이고, 여기에 마우스를 올리면 색만 진해면서 호밍이 될 뿐 클릭하거나 뭔 짓을 해도 홈페이지가 바뀌는게 없음. 그리고 지금 우리가 개발해야하는건 최종적으로
1. 챗봇 시스템
2. 사용자 집 위치를 기반으로 주변 보건소나 산후조리원 검색
3. 사용자 집 위치를 기반으로 주변 산부인과나 병원 검색
대충 이정도가 필요로 하는데 지금은 단순히 회원 기능이 없어서 챗봇 시스템을 통해 주고 받은 내용이 남아있지 않아서 새로고침 시 그전에 나눴던 채팅이 모두 사라지는 상황, 그리고 집 위치 기반으로 주변 보거너소나 산후조리원 병원, 산부인과도 현재는 직접 위치를 입력해야하는 상황임. 이런 번거로움을 해결하고자함.

# 문제상황

회원 기능이 없기 때문에 생기는 문제점들
1. 챗봇에서 전에 나눴던 대화를 불러올 수 없음
2. 자신의 위치 기반 주변 보건소나 산후조리원, 산부인과나 병원을 검색할 때 마다 자신의 위치를 입력해야한다는 번거로움이 있음


# 요구사항
1. 회원가입
	1.  사용할 아이디, 비밀번호, 집주소, 이름, 나이를 기본적으로 받고, 추가적으로 임신중이라면 임신주기, 자녀수, 소득분위 정도만 요구하는 회원가입 기능을 구현 후 해당 입력값을 바탕으로 해쉬값을 생성하여 그 사람 고유의 내부 식별용 id를 생성.
	2. data/user 디렉토리에 해당 고유 내부 식별용 id로 파일을 만들어 그안에 json을 통해 해당 유저의 정보들을 관리. (어차피 프로토타입이라 보안 신경안써도 됨)
	3. 일단 회원가입 시 자동적으로 만든 아이디와 비밀번호로 자동적으로 로그인
2. 로그인
	1. 상단 네게이션 바에 로그인 부분을 클릭시 로그인 창으로 이동함
	2. 해당 로그인 창에는 아이디, 비밀번호를 입력하는 로그인창임
	3. 로그인창에 아이디와 비밀번호를 입력하게 되면 data/user/에 있는 모든 유저을 돌아다니면서 맞는 아이디와 비밀번호가 있는지 검색(프로토타입이라 효율성 고려 안함)
	4. 맞는 아이디랑 비밀번호가 없으면 해당되는 아이디가 없다고 알려줌
	5. 맞는 아이디랑 비밀번호가 있을 시 로그인을 로그인이 성공됨
	6. 로그인 성공시 기존 네비게이션바에 흰색부분의 로그인 부부인이 호버 됐을때랑 같은색상으로 로그아웃으로 표시 됨.
	7. 해당 로그아웃을 클릭시 다시 원상태로 로그인 상태로돌아감.
